68 Chapter 4: Comments Attributions and Bylines /* Added by Rick */ Source code control systems are very good at remembering who added what, when. There is no need to pollute the code with little bylines. You might think that such com- ments would be useful in order to help others know who to talk to about the code. But the reality is that they tend to stay around for years and years, getting less and less accurate and relevant. Again, the source code control system is a better place for this kind of information. Commented-Out Code Few practices are as odious as commenting-out code. Don’t do this!
