162 Chapter 11: Systems We deﬁned an interface Bank, which will be wrapped by the proxy, and a Plain-Old Java Object (POJO), BankImpl, that implements the business logic. (We will revisit POJOs shortly.) The Proxy API requires an InvocationHandler object that it calls to implement any Bank method calls made to the proxy. Our BankProxyHandler uses the Java reﬂection API to map the generic method invocations to the corresponding methods in BankImpl, and so on. There is a lot of code here and it is relatively complicated, even for this simple case. 10 Using one of the byte-manipulation libraries is similarly challenging.
