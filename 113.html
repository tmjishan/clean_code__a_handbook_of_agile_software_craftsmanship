If one function calls another, they should be vertically close, and the caller should be above the callee, if at all possible. This gives the program a natural ﬂow. If the convention is followed reliably, readers will be able to trust that function deﬁni- tions will follow shortly after their use. Consider, for example, the snippet from FitNesse in Listing 5-5. Notice how the topmost function calls those below it and how they in turn call those below them. This makes it easy to ﬁnd the called functions and greatly enhances the readability of the whole module.
